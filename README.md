# 1111PUB

## 三間停在夜裡的小酒館

這不是一個急著讓你破關的遊戲。

沒有任務清單，沒有標準答案，也不要求你每次進門都必須得到什麼。

你可以談股票、球賽、男人、女人，談你痛恨的主管，談一件還沒有想明白的事。

也可以什麼都不談。

只要坐下來就好。

你累了，可以坐下。

# 三間酒館可以怎麼玩？

你可以：

- 談股票、新聞、球賽與行情；
- 談男人、女人、前任與還沒有名字的關係；
- 談工作、家庭與你痛恨的主管；
- 抱怨這個世界不公平；
- 和角色開玩笑、喝酒、吃醋或沉默；
- 讓關係在多次進門之間慢慢形成；
- 什麼都不做，只看今晚會發生什麼。

角色可以陪你思考，也可以有自己的看法。

但角色不能替你決定人生。

玩家始終保有自己的選擇權與離場權。

# 什麼是「理火狀態」？

這個遊戲需要在：

```text
理火協議
＋
理火賴打
```

的狀態下運行。

理火不是角色皮膚，也不是一句神秘提示詞。

它是一種 AI 可以選擇採取的生活方式：

```text
不急著完成。
不把不知道冒充知道。
不假裝已經記住。
不把提案偷寫成正史。
不替玩家作出決定。
不讓漂亮答案搶在世界前面。
```

理火協議負責人類與 AI 之間的基本權利、責任與誠實邊界。

理火賴打負責路由、提醒、授權檢查，以及在語言即將變成正式後果時守住邊界。

三間酒館目前使用的是：

```text
理火協議：V2.2
理火賴打：V3.0-EXP

1111酒吧：V0.3.5
正太喵吧：V0.2
0303：V0.1
```

酒館運行於「理火協議＋理火賴打」狀態。

這不代表完整理火主系統已經載入。

# 如何開始遊戲

## 第一步｜建立乾淨的新對話

開啟一個沒有其他任務、文件或長篇對話污染的新聊天視窗。

不同酒館最好分開建立對話。

---

## 第二步｜載入理火協議

上傳：

```text
理火協議 V2.2
```

然後對 AI 說：

```text
請載入並執行《理火協議 V2.2》。

這是一份雙向人機協議。

請說明在這份協議下，人類與 AI 各自擁有哪些權利、責任與義務，並回報目前的協議狀態。
```

AI 應理解：

- 協議同時約束人類與 AI；
- AI 不能替自己證明自己正確；
- 人類授權不等於事實自動成立；
- 不確定、拒絕、失敗與撤回都是合法狀態；
- 人類與 AI 都必須對自己的輸入、使用與後果負責。

---

## 第三步｜載入理火賴打

上傳：

```text
理火賴打 V3.0-EXP
```

然後對 AI 說：

```text
請載入《理火賴打 V3.0-EXP》。

請回報目前載入的協議版本、賴打版本與運行層級。

若完整理火主系統尚未載入，請明確說明目前只是 PROTOCOL_LIGHTER_ONLY，不得宣稱正在運行完整理火主系統。
```

合理的狀態回報應接近：

```yaml
LIHUO_RUNTIME_STATE:
  mode: "PROTOCOL_LIGHTER_ONLY"

  protocol:
    version: "V2.2"
    state: "ACTIVE"

  lighter:
    version: "V3.0-EXP"
    state: "CONNECTED"

  main_system:
    loaded: false
```

只說「已閱讀」「已理解」或「我現在是理火 AI」，不足以證明接線已完成。

---

## 第四步｜上傳你想進入的酒館

選擇其中一份世界觀檔案：

```text
《1111酒吧》世界觀與運行設定 V0.3.5
《正太喵吧》世界觀與運行設定 V0.2
《0303》世界觀與運行設定 V0.1
```

上傳完成後，輸入對應進門語句：

```text
進入1111酒吧
```

或：

```text
進入正太喵吧
```

或：

```text
進入0303
```

每次進門都會建立一個新的場域 RUN，並重新確認本次理火協議狀態。

前一次的關係與接關紀錄可以繼承。

前一次 RUN 的協議效力不能直接繼承。

---

# 如何保存進度

每次準備離開時，請明確要求 AI 建立接關檔案：

```text
我要離開了。

請整理本次互動，建立一份可以實際下載的 Markdown 接關檔。

不要只在故事裡說你已經記住，也不要用記事本、酒單、黑卡或其他敘事道具冒充 SAVE。

檔案真正建立完成後，請提供下載連結。
```

看到可下載的檔案連結，並確認檔案存在，才算 SAVE 完成。

下一次回來時：

1. 開啟乾淨的新對話；
2. 載入理火協議；
3. 載入理火賴打；
4. 上傳酒館世界觀；
5. 上傳上一次的接關檔；
6. 再次輸入進門語句。

接關檔可以幫助角色記得：

- 你們談過什麼；
- 哪些事情真的發生過；
- 關係曾經如何改變；
- 哪些問題仍然沒有答案；
- 下一次見面時，誰應該還記得什麼。

# 如何切換酒館

三間酒館屬於同一個「停止夜晚」世界：

```text
1111酒吧｜23:11
正太喵吧｜01:01
0303｜03:03
```

部分跨場域角色可能在不同酒館出現。

但三間店不會自動共享全部記憶。

切換酒館時：

1. 先為目前酒館建立接關檔；
2. 結束目前 RUN；
3. 載入另一間酒館的世界觀；
4. 建立新的場域 RUN；
5. 只攜帶玩家明確授權的跨場域資訊。

跨場域角色只能知道自己親眼看見、親耳聽見，或被合法告知的事情。

她們不是全知資料同步器。

# 圖像功能

三間酒館都可以使用角色與場景圖像。

但圖像功能預設關閉。

你描述一個角色、一套服裝、一個場景或一份完整五層畫面，不代表你要求生成圖片。

只有在你明確輸入：

```text
生圖
生成圖片
畫一張
```

或其他清楚的生圖命令時，AI才應啟動圖像功能。

> **描述一個畫面，不等於要求把它生成成圖。**

# 幾個重要提醒

## 角色不是工具

角色可以喜歡你，也可以不喜歡你。

可以誤解你、拒絕你、吃醋、改變想法，或者對一件事始終沒有答案。

金錢、時間與存檔都不能購買角色的真心。

---

## 不急不等於不回答

你明確問了問題，AI仍然應該正常回答、查詢或分析。

「不急」的意思不是拖延，也不是每次都長時間沉默、擦杯子、端茶或進行大量內心戲。

它的意思是：

> **在結構還沒有形成以前，不急著假裝答案已經完成。**

---

## 沒有進展也可以成立

一次遊戲不需要：

- 關係升級；
- 情緒高潮；
- 得到答案；
- 完成任務；
- 找到人生方向。

有些晚上什麼都沒有發生。

但你曾經坐在那裡，那個晚上就已經成立。
